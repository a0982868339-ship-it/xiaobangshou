# 小帮手项目 - 完整测试方案

## 📋 测试概述

**项目名称**: 小帮手 - 本地生活服务O2O平台  
**测试目标**: 确保系统功能完整、性能稳定、用户体验良好  
**测试环境**: 开发环境（本地）  
**测试周期**: 建议5-7个工作日  

---

## 🎯 测试范围

### 系统组成
1. **后端API** (Node.js + Express) - http://localhost:3000
2. **移动端H5-用户端** (Vue3 + Vant) - http://localhost:9000
3. **移动端H5-服务者端** (Vue3 + Vant) - http://localhost:9001
4. **Web管理后台** (Vue3 + Element Plus) - http://localhost:8080
5. **数据库** (MySQL 8.0)
6. **缓存** (Redis)

---

## 📊 测试策略

### 测试类型占比
- 功能测试: 50%
- 接口测试: 20%
- 兼容性测试: 15%
- 性能测试: 10%
- 安全测试: 5%

### 测试优先级
- **P0（核心功能）**: 注册登录、服务浏览、订单创建、支付流程
- **P1（重要功能）**: 个人中心、地址管理、订单管理、会员套餐
- **P2（一般功能）**: 搜索、筛选、分享、评价
- **P3（次要功能）**: 客服咨询、帮助中心、关于我们

| 用例ID | 测试项 | 请求方法 | 接口路径 | 优先级 |
|--------|--------|----------|----------|--------|
| API-001 | 发送短信验证码 | POST | /api/auth/sms/send | P0 |
| API-002 | 验证码登录/注册 | POST | /api/auth/sms/login | P0 |
| API-003 | 密码登录 | POST | /api/auth/login | P0 |
| API-004 | 获取用户信息 | GET | /api/auth/me | P0 |
| API-005 | 退出登录 | POST | /api/auth/logout | P1 |

#### 详细测试步骤

**API-001: 发送短信验证码**
```bash
# 正常场景
curl -X POST http://localhost:3000/api/auth/sms/send \
  -H "Content-Type: application/json" \
  -d '{"phone":"13800138000"}'

# 预期结果: 返回200，提示发送成功

# 异常场景
# 1. 手机号格式错误
curl -X POST http://localhost:3000/api/auth/sms/send \
  -H "Content-Type: application/json" \
  -d '{"phone":"12345"}'
# 预期: 返回400，提示手机号格式错误

# 2. 频繁请求（60秒内重复发送）
# 快速执行两次上述命令
# 预期: 第二次返回429，提示请勿频繁发送
```

**API-002: 验证码登录**
```bash
curl -X POST http://localhost:3000/api/auth/sms/login \
  -H "Content-Type: application/json" \
  -d '{"phone":"13800138000","code":"123456"}'

# 预期结果: 返回token和用户信息
```

**API-003: 密码登录**
```bash
curl -X POST http://localhost:3000/api/auth/login \
  -H "Content-Type: application/json" \
  -d '{"phone":"13800138000","password":"123456"}'
```

**API-004: 获取用户信息（需要token）**
```bash
curl -X GET http://localhost:3000/api/auth/me \
  -H "Authorization: Bearer YOUR_TOKEN_HERE"
```
