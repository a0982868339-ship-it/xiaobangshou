### 问题4: 订单状态更新失败

**现象**:
```
Error: Field 'balance_before' doesn't have a default value
```

**原因**:
- 插入wallet_records表时缺少必需字段
- balance_before和balance_after字段为NOT NULL但未提供值

**修复方案**:
```javascript
// 修改orderController.js
// 在插入钱包流水前先获取当前余额
const [users] = await connection.query('SELECT balance FROM users WHERE id = ?', [providerUserId]);
const balanceBefore = parseFloat(users[0].balance || 0);
const balanceAfter = balanceBefore + parseFloat(earnings);

// 插入时包含这些字段
await connection.query(
  `INSERT INTO wallet_records (user_id, order_id, type, amount, balance_before, balance_after, description) 
   VALUES (?, ?, 3, ?, ?, ?, ?)`,
  [providerUserId, id, earnings, balanceBefore, balanceAfter, description]
);
```

**修复结果**: ✅ 已修复
- 订单状态更新成功
- 收益结算正常
- 钱包流水记录完整

### 问题5: 地址添加接口失败

**现象**:
- 添加地址返回400错误
- 提示"添加失败"

**原因**:
- 前端发送的字段名使用驼峰命名（contactName）
- 后端期望的字段不完整
- 缺少对detail和isDefault字段的支持

**修复方案**:
```javascript
// 修改addressController.js
exports.addAddress = async (req, res) => {
  const { contactName, contactPhone, address, detail, province, city, district, isDefault } = req.body;
  const addressDetail = address || detail; // 兼容两种字段名
  const is_default = isDefault !== undefined ? isDefault : 0;
  
  // 如果设置为默认地址，先将其他地址设为非默认
  if (is_default === 1) {
    await db.query('UPDATE user_addresses SET is_default = 0 WHERE user_id = ?', [req.user.id]);
  }
  
  // 插入新地址
  await db.query(...);
}
```

**修复结果**: ✅ 已修复
- 地址添加成功
- 默认地址逻辑正常
