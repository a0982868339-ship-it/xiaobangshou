## 🎯 功能说明

### 三层设置结构

```
第一层：城市选择
  └─ 常驻城市: 从全国城市中选择（北京、上海、广州、深圳等）

第二层：区县选择
  └─ 覆盖全城 OR 选择多个区县
     ├─ 覆盖全城（默认）
     └─ 或选择: 朝阳区、海淀区、东城区...（可多选）

第三层：距离限制
  └─ 最远接单距离: 5-50公里（可选，步进5公里）
```

---

## 📱 使用流程

### 步骤1：选择常驻城市

```
1. 进入"服务区域"页面
2. 点击"选择城市"
3. 选择省份（如：北京市）
4. 选择城市（如：北京市）
5. 点击"确认"
```

**支持的城市**：
- 北京市（16个区）
- 上海市（16个区）
- 广州市（11个区）
- 深圳市（9个区）
- 其他城市（使用@vant/area-data，全国覆盖）

---

### 步骤2：设置接单范围

#### 选项A：覆盖全城（默认）

```
1. 开启"覆盖全城"开关
2. 接受该城市所有区县的订单
3. 适合：愿意跑远的服务者
```

#### 选项B：选择特定区县

```
1. 关闭"覆盖全城"开关
2. 点击"选择区县"
3. 勾选您愿意服务的区县（可多选）
   ☑️ 朝阳区
   ☑️ 海淀区
   ☑️ 东城区
4. 点击"确定"
5. 已选区县显示为绿色Tag
6. 可点击Tag右侧X删除
```

**示例场景**：
- 住在朝阳区 → 只选朝阳区、通州区（邻近）
- 住在海淀区 → 只选海淀区、西城区（邻近）

---

### 步骤3：设置距离限制（可选）

```
1. 点击"最远接单距离"
2. 使用步进器调整：5公里、10公里、15公里...
3. 最大50公里
```

**作用**：
- 即使在接单区县内，超过此距离的订单也不推送
- 例如：选了朝阳区，设置15公里
  - 望京的订单 → 推送（10公里内）
  - 亦庄的订单 → 不推送（20公里外）

---

### 步骤4：保存设置

```
1. 点击底部"保存设置"按钮
2. 提示"保存成功"
3. 自动返回上一页
```

## 🔧 技术实现

### 前端

**使用的组件**：
- `van-area` - 城市选择（全国数据）
- `van-checkbox-group` - 区县多选
- `van-stepper` - 距离调整
- `van-tag` - 区县展示

**数据来源**：
- `@vant/area-data` - 全国省市区数据（已安装）
- 内置主要城市区县数据（北上广深）

---

### 后端

**接口**：
```
PUT /api/providers/service-areas

Body:
{
  "serviceAreas": {
    "city": "北京市",
    "cityCode": "110100",
    "allCity": false,
    "districts": [
      {"code": "110105", "name": "朝阳区"},
      {"code": "110108", "name": "海淀区"}
    ],
    "maxDistance": 15
  }
}

Response:
{
  "success": true,
  "message": "服务区域更新成功"
}
```

**数据存储**：
- 存储在 `providers` 表的 `service_areas` 字段（JSON格式）
