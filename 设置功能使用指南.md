## 📱 用户端设置功能

### 访问方式
```
1. 打开用户端: http://localhost:9000
2. 点击底部导航"我的"
3. 点击"设置"进入设置页面
```

### 功能列表

#### 1. **退出登录** ✅
- **位置**: 设置页面底部红色按钮
- **功能**: 清除登录信息，跳转登录页
- **特点**: 二次确认，防止误操作

#### 2. **个人信息编辑** ✅
- **路径**: 设置 → 个人信息
- **功能**:
  - 修改头像（开发中）
  - 修改昵称
  - 选择性别
  - 填写真实姓名
- **测试**: 修改后立即生效

#### 3. **账户安全** ✅
- **路径**: 设置 → 账户安全
- **功能**:
  - 修改密码
  - 绑定手机号（开发中）
  - 实名认证（开发中）
  - 登录设备管理（开发中）
  - 账号注销（开发中）

#### 4. **修改密码** ✅
- **路径**: 设置 → 账户安全 → 修改密码
- **功能**:
  - 验证旧密码
  - 设置新密码（至少6位）
  - 修改成功后需重新登录
- **测试**: 
  ```
  旧密码: 123456
  新密码: 123456
  确认密码: 123456
  ```

#### 5. **通知设置** ✅
- **功能**:
  - 订单通知开关
  - 活动通知开关
  - 系统消息开关
- **特点**: 实时保存，本地存储

#### 6. **清除缓存** ✅
- **功能**: 清除应用缓存
- **特点**: 保留登录信息和用户数据

#### 7. **其他功能**
- 地址管理（已有）
- 帮助中心（开发中）
- 意见反馈（开发中）
- 关于我们

## 🔌 后端接口

### 新增接口列表

#### 1. 修改密码
```
POST /api/users/change-password
Authorization: Bearer {token}

Body:
{
  "oldPassword": "123456",
  "newPassword": "newpass123"
}

Response:
{
  "success": true,
  "message": "密码修改成功"
}
```

#### 2. 更新个人信息
```
PUT /api/users/profile
Authorization: Bearer {token}

Body:
{
  "nickname": "新昵称",
  "gender": 1,
  "realName": "张三",
  "avatar": "头像URL"
}

Response:
{
  "success": true,
  "message": "更新成功"
}
```

#### 3. 服务者切换接单状态
```
PUT /api/providers/toggle-status
Authorization: Bearer {token}

Body:
{
  "status": 1  // 0-休息中, 1-接单中
}

Response:
{
  "success": true,
  "message": "已开启接单"
}
```

#### 4. 获取服务者状态
```
GET /api/providers/status
Authorization: Bearer {token}

Response:
{
  "success": true,
  "data": {
    "status": 1
  }
}
```
