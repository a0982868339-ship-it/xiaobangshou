## 🔧 问题修复记录

### 问题 1: 地址添加接口失败

**问题描述**:
- 接口返回400错误
- 提示"添加失败"

**原因分析**:
- 前端发送的字段名使用驼峰命名（contactName）
- 后端控制器期望的字段名不完整
- 缺少对 `detail` 字段的支持
- 缺少对 `isDefault` 字段的处理

**修复方案**:
```javascript
// 修改 backend/src/controllers/addressController.js
exports.addAddress = async (req, res) => {
  try {
    const { contactName, contactPhone, address, detail, province, city, district, isDefault } = req.body;
    const addressDetail = address || detail; // 兼容两种字段名
    const is_default = isDefault !== undefined ? isDefault : 0;
    
    // 如果设置为默认地址，先将其他地址设为非默认
    if (is_default === 1) {
      await db.query('UPDATE user_addresses SET is_default = 0 WHERE user_id = ?', [req.user.id]);
    }
    
    const [result] = await db.query(
      'INSERT INTO user_addresses (user_id, contact_name, contact_phone, address, province, city, district, is_default) VALUES (?, ?, ?, ?, ?, ?, ?, ?)',
      [req.user.id, contactName, contactPhone, addressDetail, province, city, district, is_default]
    );
    sendSuccess(res, { id: result.insertId }, '添加成功');
  } catch (error) {
    console.error('添加地址错误:', error);
    sendError(res, '添加失败: ' + error.message);
  }
};
```

**修复结果**: ✅ 已修复并测试通过
